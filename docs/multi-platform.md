# 多端打包说明

## 当前结论

Minen Studio 是支持 Agnes AI API 的独立跨端 Web 应用：桌面端使用侧栏，移动端使用底部导航，并提供 `app/manifest.ts` 让浏览器识别为 PWA。

但它目前不是纯静态应用，因为聊天、图片、视频都依赖 Next.js Route Handler：

- `/api/chat`
- `/api/image`
- `/api/video/create`
- `/api/video/query`
- `/api/video/status/[taskId]`

这些接口会读取请求头或环境变量中的 API Key，并代理访问上游 Agnes API。Next.js 静态导出不支持这种动态服务端逻辑，所以不要直接配置 `output: 'export'`。

## 推荐路线

### Web/PWA

部署完整 Next.js 应用。用户可以直接在浏览器访问，也可以通过浏览器的“安装应用”能力添加到桌面或手机主屏幕。

适合：

- 最快上线
- 同一套代码覆盖桌面和移动浏览器
- 后续继续保留服务端代理能力

### 桌面端

使用 Electron 或 Tauri 包一个桌面壳。壳应用可以加载线上地址，也可以随应用启动本地 Next.js 服务。

推荐优先加载线上地址，这样更新成本最低。

### 移动端

使用 Capacitor 包装线上 PWA 地址。这样可以复用当前页面、状态管理和接口逻辑。

如果要进一步接近原生体验，再补：

- 完整 192/512 PNG 图标
- 启动图
- Service Worker 离线策略
- 系统分享、文件保存等原生能力

### 纯静态托管

只有在移除或外置 `/app/api/*` 后才建议做。可选方案：

- 前端直接请求 Agnes API，但 API Key 会暴露在客户端。
- 拆出独立后端服务，前端静态导出后请求独立后端。

## 后续优化清单

- 增加真实 PWA PNG 图标，而不是只复用 favicon。
- 增加 Service Worker，缓存应用壳和静态资源。
- 把媒体 URL 历史迁到后端数据库，避免用户换设备后历史丢失。
- 为图片和视频下载增加服务端转存能力，统一保存到对象存储。
- 增加 Playwright 冒烟测试，覆盖聊天、生图、视频、设置四个页面的基础加载。
